## 1. 系统架构概览

**当前架构**：**Python 后端 + 纯静态前端 + JSON 文件存储**

不再是之前的纯静态架构。当前系统引入了 Python 后端服务器，负责：
- 提供 HTTP 访问（静态文件）
- 提供 REST API（供后台管理用）
- 实现内存缓存（加速首页加载）
- 支持分类/标签拖拽排序

### 文件结构

```
blog/
├── index.html              # 博客首页入口
├── admin.html              # 管理后台入口（需要登录）
├── server.py               # Python 后端服务器
├── start_server.sh         # 启动/停止脚本
├── 使用指南.md             # 本文档
├── css/
│   ├── style.css           # 博客前台样式
│   └── admin.css           # 管理后台样式（含登录页样式）
├── js/
│   ├── app.js              # 博客前台核心逻辑
│   └── admin.js            # 管理后台逻辑（含登录认证）
├── data/                   # 数据目录
│   ├── posts.json          # 文章数据
│   ├── categories.json     # 分类数据
│   ├── tags.json           # 标签数据
│   ├── users.json          # ⭐ 用户账号数据（SHA-256 加盐哈希存储密码）
│   └── backups/            # 文章自动备份
└── images/                 # 图片目录（可放文章配图）
```

### 工作原理

```
┌────────────── 浏览器 ──────────────┐
│  index.html  ─►  app.js            │
│  admin.html  ─►  admin.js          │
│      │                              │
│      ▼ AJAX 请求                    │
│  /api/posts      (文章列表)         │
│  /api/categories (分类列表)         │
│  /api/tags      (标签列表)         │
│  /api/bootstrap (聚合数据，加速加载)│
│  /api/stats     (统计数据)         │
└──────┬─────────────────────────────┘
       ▼
  ┌──────────────────┐
  │   server.py      │  Python HTTP 服务器
  │  (REST API + 静态文件)│  含内存缓存 + JSON 存储
  └──────┬───────────┘
         ▼
  ┌──────────────────┐
  │  data/*.json     │  posts.json / categories.json / tags.json
  └──────────────────┘
```

### 🔑 重要结论（哪些操作需要重启服务器）

| 操作 | 是否需要重启服务器 |
|------|-------------------|
| 通过后台 (`admin.html`) 新增/修改/删除文章 | ❌ **不需要**，后台直接调用 API，立即生效 |
| 通过后台修改分类/标签/拖拽排序 | ❌ **不需要** |
| 修改 `posts.json`、`categories.json`、`tags.json` 等 JSON 文件 | ❌ **不需要**，刷新浏览器即可 |
| 修改 `style.css`、`admin.css`（样式） | ❌ **不需要**，刷新浏览器即可 |
| 修改 `app.js`、`admin.js`（逻辑） | ❌ **不需要**，刷新浏览器即可 |
| 修改 `server.py` 或修改端口号 | ✅ **需要**，`bash start_server.sh stop` 然后再启动 |

**注意**：修改文本文件后，建议按 `Ctrl+Shift+R` 强制刷新浏览器，避开本地缓存。

## 3. 使用博客前台

服务器启动后，浏览器访问 `http://localhost:8080/`。

### 3.1 页面导航

博客采用 URL hash 路由（`#/home`、`#/categories` 等），共 5 个页面：

| 路由 | 页面内容 |
|------|---------|
| `#/home` | 首页 — 文章列表 + 侧栏（作者卡片、分类、标签） |
| `#/post/<id>` | 文章详情页 — 完整 Markdown 文章内容 |
| `#/categories` | 分类页 — 所有分类及各分类文章数 |
| `#/tags` | 标签页 — 所有标签及各标签文章数 |
| `#/archive` | 归档页 — 按年份归档 |
| `#/about` | 关于页 |

### 3.2 数据加载逻辑（供参考）

`app.js` 按以下优先级加载数据：

1. **第一优先级**：请求 `/api/bootstrap`（聚合 API，一次拿到 posts + categories + tags，**最快**）
2. **回退方案 A**：并行请求 `/api/posts`、`/api/categories`、`/api/tags`
3. **回退方案 B**：直接读取 `data/posts.json`、`data/categories.json`、`data/tags.json`（JSON 文件）
4. **最终兜底**：空数组 + 显示"服务器未启动"图文引导页

所以：
- **服务器正常运行**：首页加载非常快（一次请求 + 内存缓存）
- **服务器未运行**：前端会显示"服务器未启动"引导页，而不是白屏报错

## 5. 管理分类与标签（支持拖拽排序）

分类和标签现在是**独立数据**，不再仅从文章中推导。你可以：
- 创建分类/标签（含自定义颜色、图标、描述）
- 编辑已有分类/标签
- **拖拽排序** — 用鼠标直接拖动，松手即保存，首页立即按新顺序呈现

### 5.1 分类管理

1. 后台左侧菜单点击「📂 分类管理」
2. 页面显示所有分类卡片（每张卡片显示自定义图标、颜色、文章数）
3. **新增分类**：点击右上角「➕ 新建分类」
   - 名称：分类中文名，如 "Linux"
   - 描述：可选，分类简介
   - 颜色：点击色块选择，将用于分类标签背景
   - 图标：输入 Emoji 作为图标，如 🐧、💻、📚
4. **编辑/删除**：点击卡片上对应按钮
5. **拖拽排序**（⭐ 新功能）：
   - 鼠标按住某张分类卡片（任意位置）
   - 拖到目标位置（其他卡片上有蓝色高亮表示目标位）
   - **松手即保存**，首页侧栏和分类页立即按新顺序排列

### 5.2 标签管理

1. 后台左侧菜单点击「🏷 标签管理」
2. 操作与分类管理类似，每张标签卡片：
   - 显示标签颜色（作为小色块）
   - 显示使用该标签的文章数
3. **新增/编辑/删除**：同分类
4. **拖拽排序**：同样支持鼠标直接拖动排序

### 5.3 排序生效原理

- 分类/标签的每条数据包含 `sortOrder` 字段（数字）
- 前端页面渲染时按 `sortOrder` 升序排列
- 拖拽后后端会把目标位置的 `sortOrder` 重新编号
- 新创建的分类/标签默认排到末尾

### 5.4 仍支持手动编辑 JSON（高级）

如你喜欢手动编辑，数据格式如下：

## 6. 服务器未启动时的处理

**当前系统的优化**：服务器未运行时，不会白屏或无限加载，而是显示友好的引导页，告诉你怎么启动服务器。

### 6.1 博客首页的表现

当你打开 `http://localhost:8080/` 但服务器没运行时：
- 看到「⚠️ 博客服务器未启动」大标题
- 看到 4 步启动指引 + 一键「🔄 重新加载」按钮
- 按指引在终端执行：
  ```bash
  cd /home/xys/Linux-C-Cplus-Project/blog
  bash start_server.sh stop   # 先清理旧进程（如果有）
  bash start_server.sh         # 启动新服务器
  ```
- 服务器启动后，点击页面上的「🔄 重新加载」或手动刷新浏览器

### 6.2 管理后台的表现

当你打开 `admin.html` 但服务器没运行时：
- 看到「⚠️ 管理后台服务器未启动」引导页
- 同样给出命令行启动步骤
- **服务器未启动时，后台的新增/修改/删除操作都不可用**（因为需要调用 API）

### 6.3 检查服务器是否在运行

```bash
# 看端口
ss -tlnp 2>/dev/null | grep 8080 && echo "✅ 8080 端口有服务" || echo "⚠️ 8080 无服务"

# 直接请求 API
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8080/api/bootstrap
# 返回 200 表示服务器运行正常；返回 000 或报错表示未启动

# 看进程
ps aux | grep "server.py" | grep -v grep
```

## 7. 账号与安全

> ⭐ **重要更新**：管理后台现在需要登录才能访问。

### 7.1 默认账号

首次启动服务器时，系统会自动创建默认管理员账号：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| `admin` | `admin123` | 管理员 |

> ⚠️ **强烈建议**：首次登录后立即修改密码！

### 7.2 登录管理后台

1. 打开浏览器访问 `http://localhost:8080/admin.html`
2. 看到登录页面（紫蓝色渐变背景）
3. 输入用户名和密码，点击「登录」
4. 登录成功后自动进入管理后台主界面
5. 顶部导航栏右侧新增「🚪 退出登录」按钮

### 7.3 修改密码

1. 在后台左侧菜单点击「⚙️ 系统设置」
2. 滚动到「🔐 账号安全」区块
3. 填写旧密码、新密码（至少 6 位）和确认密码
4. 点击「💾 修改密码」

> 注意：修改密码后，所有现有会话（包括其他浏览器标签页）会自动失效，需要重新登录。

### 7.4 登出

点击顶部导航栏右侧的「🚪 退出登录」，系统会：
- 清除服务器端会话
- 清除浏览器本地存储的登录凭证
- 自动跳转回登录页面

### 7.5 会话机制

- 登录成功后，服务器生成一个 64 字符的安全随机 Token
- Token 存储在浏览器 `localStorage` 中
- 每个 API 请求通过 `Authorization: Bearer <token>` header 携带 Token
- 登出或修改密码后，当前 Token 立即失效
- 会话数据存储在服务器**内存**中，服务器重启后所有会话失效（用户需要重新登录）

### 7.6 账号数据存储

用户账号数据存储在 `blog/data/users.json`：

```json
[
  {
    "username": "admin",
    "passwordHash": "salt$hash（SHA-256 加盐哈希，不存储明文密码）",
    "role": "admin",
    "createdAt": "2026-06-10T00:00:00.000000"
  }
]
```

### 7.7 忘记密码怎么办？

如果忘记了密码，可以手动重置：

```bash
# 方法 1：删除 users.json，重启服务器自动重建
cd /home/xys/Linux-C-Cplus-Project/blog/data
rm users.json
# 然后重启服务器：
bash start_server.sh stop && bash start_server.sh
# 账号恢复为 admin / admin123
```

### 7.8 API 认证接口（供参考）

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/auth/check` | 检查登录状态 |
| `POST` | `/api/auth/login` | 登录（无需 token） |
| `DELETE` | `/api/auth/logout` | 退出登录 |
| `PUT` | `/api/auth/change-password` | 修改密码 |

## 8. 资源管理（图片/文件上传与插入）

> ⭐ **新增功能**：后台内置完整的文件上传与管理功能，无需手动复制文件到目录。

### 8.1 功能入口

登录管理后台后，左侧菜单新增「📎 资源管理」。

### 8.2 上传文件

进入「资源管理」页面，可以看到上传区：

1. **拖拽上传**：直接将文件拖到上传框中
2. **点击上传**：点击上传区，弹出文件选择对话框
3. 支持多文件同时上传

支持的文件类型（自动分类）：

| 分类 | 文件类型 |
|------|---------|
| 📷 图片 | png, jpg, jpeg, gif, webp, svg, bmp, ico |
| 📄 文档 | pdf, doc, docx, xls, xlsx, ppt, pptx, txt |
| 🎬 视频 | mp4, webm, ogg, avi, mov |
| 🎵 音频 | mp3, wav, ogg, flac, aac |
| 📦 压缩包 | zip, rar, 7z, tar, gz |
| 💻 代码 | cpp, c, h, hpp, py, js, html, css, json, xml, md |
| 📋 其他 | 其他格式 |

单个文件最大 50MB。

### 8.3 管理文件

资源管理页面显示所有已上传文件（按上传时间倒序）：

- **分类筛选**：点击顶部的分类标签（📷 图片 / 📄 文档 等）过滤
- **搜索**：输入文件名关键词搜索
- **插入到文章**：点击卡片上的「插入」按钮，自动以 Markdown 格式插入到当前文章正文中
  - 图片插入格式：`` ![文件名](url) ``
  - 其他文件插入格式：`` [📎 文件名](url) ``
- **删除**：点击「删除」，物理文件同步删除（不可恢复）

### 8.4 在文章编辑器中快速插入

在新建/编辑文章时，正文输入框工具栏新增「🖼 插入图片」按钮：
- 点击后自动切换到「资源管理」tab
- 选择资源后点击「插入」，Markdown 自动插入到光标位置

### 8.5 资源数据结构（resources.json）

```json
    "category": "image",
    "uploadedBy": "admin",
    "uploadedAt": "2026-06-10T01:00:00.000000"
  }
]
```

### 8.6 资源相关 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/upload` | 上传文件（需登录，multipart/form-data） |
| `GET` | `/api/resources` | 获取资源列表（需登录） |
| `GET` | `/api/resources?category=image` | 按分类筛选资源 |

| `DELETE` | `/api/resources/{id}` | 删除资源（需登录，同步删除物理文件） |

> 注意：所有文件通过 `/uploads/文件名` 路径直接访问（无需登录），适合在文章正文中引用。

## 9. 公网部署完整指南

> ⚠️ **安全警告**：本博客系统的管理后台没有内置身份验证（登录/密码）。
> 一旦部署到公网，任何访问到 `admin.html` 的人都可以修改你的文章/分类/标签。
> 建议在公网部署前，先对 `admin.html` 做访问控制（见 9.4 节）。

### 11.1 方案总览

| 方案 | 成本 | 难度 | 适用场景 |
|------|------|------|----------|
| **方案 A：云服务器 + Nginx 反向代理** | 约 30-100 元/月 | ⭐⭐ | 已有云服务器，希望完全掌控（支持后台在线编辑） |
| **方案 B：静态托管（GitHub Pages 等）** | 免费 | ⭐ | 只对外展示，不在公网后台编辑 |
| **方案 C：内网穿透（frp / cloudflared）** | 免费或少量费用 | ⭐⭐ | 家里有常开机的电脑/树莓派 |
| **方案 D：纯 Python 公网暴露** | 0（不推荐） | ⭐ | 临时测试使用 |

### 9.2 方案 A：云服务器 + Nginx 反向代理（推荐）

此方案让你**在公网也能使用管理后台**（通过 API 调用 `server.py`）。

**前提**：有一台云服务器（阿里云 ECS、腾讯云 CVM、AWS EC2 等，Linux 系统）。

#### 步骤 1：代码上传到服务器

```bash
# 在服务器上（例如部署到 /opt/blog/）
mkdir -p /opt/blog && cd /opt/blog

# 方式 1：scp 上传（从本地执行）
scp -r /home/xys/Linux-C-Cplus-Project/blog/* user@你的服务器IP:/opt/blog/

# 方式 2：git 仓库
git clone <你的仓库地址> /opt/blog
```

#### 步骤 2：确认依赖（通常系统自带）

```bash
python3 --version   # 需要 Python 3.6+
sudo apt update && sudo apt install -y nginx   # Debian/Ubuntu
# 或
sudo yum install -y nginx                       # CentOS
```

#### 步骤 3：用 systemd 把 Python 服务器做成守护进程

创建 `/etc/systemd/system/blog.service`：

```ini
[Unit]
Description=My Blog Python Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/blog
ExecStart=/usr/bin/python3 /opt/blog/server.py
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动并设为开机自启：

```bash
sudo systemctl daemon-reload
sudo systemctl enable blog
sudo systemctl start blog
sudo systemctl status blog    # 确认 active (running)
```

服务器会监听 `0.0.0.0:8080`。

#### 步骤 4：配置 Nginx 反向代理

创建 `/etc/nginx/sites-available/blog`（或 CentOS 下 `/etc/nginx/conf.d/blog.conf`）：

```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;   # 改成你的域名；没有域名可写服务器公网IP

    # 静态文件 & API 统一转发到 Python 后端
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用：

```bash
# Debian/Ubuntu
sudo ln -s /etc/nginx/sites-available/blog /etc/nginx/sites-enabled/blog
sudo nginx -t                      # 检查配置语法
sudo systemctl reload nginx
```

#### 步骤 5：开启 HTTPS（强烈推荐）

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
# Certbot 会自动修改 Nginx 配置并开启 HTTPS 重定向
```

现在：
- 博客首页：`http(s)://你的域名/`
- 管理后台：`http(s)://你的域名/admin.html`

### 9.4 安全加固：给 admin.html 加访问密码

管理后台默认没有登录。部署到公网前，强烈建议对 `admin.html` 和 `/api/` 做访问控制。

#### 方法 1：Nginx Basic Auth（最简单，推荐）

1. 用 `htpasswd` 创建密码文件：

```bash
sudo apt install -y apache2-utils
sudo htpasswd -c /etc/nginx/.htpasswd 你的用户名
# 输入两次密码
```

2. 修改 Nginx 配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 对管理后台和 API 加密码保护
    location = /admin.html {
        auth_basic "Admin Area";
        auth_basic_user_file /etc/nginx/.htpasswd;
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /api/ {
        auth_basic "Admin Area";
        auth_basic_user_file /etc/nginx/.htpasswd;
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

3. 重新加载：`sudo nginx -t && sudo systemctl reload nginx`

之后访问 `admin.html` 或调用 API 时，浏览器会弹出用户名/密码框。

#### 方法 2：把后台文件改名（简单混淆）

把 `admin.html` 改名为一个不容易猜到的名字，例如 `my-secret-admin-2026.html`。
这只是第一道门槛，不是真正的安全措施。

#### 方法 3：只在本机/特定 IP 启用后台

```nginx
location = /admin.html {
    allow 127.0.0.1;
    allow 你自己的公网IP;
    deny all;
    proxy_pass http://127.0.0.1:8080;
}
```

### 9.5 方案 C：内网穿透（家庭服务器/树莓派）

如果博客跑在家里的电脑或树莓派上，希望外网也能访问：

#### 推荐 1：frp（免费，需要一台公网中转服务器）

1. 公网服务器（中转）运行 frps（服务端）：
   ```bash
   # 从 https://github.com/fatedier/frp/releases 下载
   tar -zxf frp_*.tar.gz && cd frp_*
   ./frps -c ./frps.toml     # 默认监听 7000/tcp
   ```

2. 家庭电脑运行 frpc（客户端），把本地 8080 映射出去：
   ```toml
   [blog]
   type = http
   local_port = 8080
   custom_domains = blog.your-domain.com
   ```

3. 浏览器访问 `http://blog.your-domain.com`。

#### 推荐 2：Cloudflare Tunnel（无需公网服务器，免费）

1. 注册 Cloudflare，添加自己的域名；
2. 在家庭电脑上安装 `cloudflared`：
   ```bash
   curl -L --output cloudflared https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
   chmod +x cloudflared && sudo mv cloudflared /usr/local/bin/
   ```
3. 登录并创建隧道：
   ```bash
   cloudflared tunnel login
   cloudflared tunnel create blog-tunnel
   cloudflared tunnel route dns blog-tunnel blog.your-domain.com
   cloudflared tunnel run --url http://localhost:8080 blog-tunnel
   ```

4. 之后访问 `https://blog.your-domain.com`。

参考：https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/

## 10. 常见问题 FAQ

### Q1: 修改了文章/分类/标签，首页没有变化？

**A1**：按以下步骤排查：
1. 确认服务器仍在运行：`curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/api/bootstrap`（应返回 `200`）
2. **浏览器硬刷新**：`Ctrl+Shift+R` 或 `Cmd+Shift+R`（避开浏览器缓存）
3. 如果仍看不到，检查 F12 Console 是否有红色错误信息

### Q2: 打开博客显示「服务器未启动」引导页？

**A2**：按页面上的指引执行两条命令即可：

```bash
cd /home/xys/Linux-C-Cplus-Project/blog
bash start_server.sh
```

服务器启动后，点击页面上的「🔄 重新加载」按钮。

### Q3: 如何在后台对分类/标签进行排序？

**A3**：进入后台的「分类管理」或「标签管理」，**用鼠标按住卡片拖到目标位置**，松手即保存。首页侧栏和分类/标签页会立即按新顺序排列。

### Q4: 如何添加图片到文章中？

**A4**：两种方式：

1. **外部图片**（推荐，无需管理文件）：
   ```markdown
   ![图片描述](https://example.com/path/to/image.png)
   ```

2. **本地图片**：
   - 放入 `blog/images/` 目录，如 `images/my-photo.png`
   - 在文章正文中引用：
     ```markdown
     ![图片描述](images/my-photo.png)
     ```

### Q5: 让文章按访问量而不是按日期排序？

**A5**：编辑 `blog/js/app.js`，找到首页渲染时的排序代码（搜索 `sort`）：

```javascript
// 按日期（当前行为）
const sortedPosts = [...posts].sort((a, b) => new Date(b.date) - new Date(a.date));

// 改为按浏览量
const sortedPosts = [...posts].sort((a, b) => b.views - a.views);
```

### Q6: 如何统计真实的文章访问量？

**A6**：当前 `views` 字段是静态数字，不会自动增加。要实现真实访问统计有三种方案：
1. 用 `localStorage` 在本地追踪（仅供个人参考，不跨用户）
2. 接入第三方统计：百度统计、Google Analytics、Umami 等
3. 扩展 `server.py`，在文章详情页打开时调用 API 自增 views 字段

### Q7: 修改了 server.py，如何让新代码生效？

**A7**：必须重启服务器：

```bash
cd /home/xys/Linux-C-Cplus-Project/blog
bash start_server.sh stop
bash start_server.sh
```

### Q8: posts.json 损坏了怎么办？

**A8**：`data/backups/posts_backup.json` 是每次保存前的自动备份。用备份文件覆盖即可：

```bash
cp /home/xys/Linux-C-Cplus-Project/blog/data/backups/posts_backup.json \
   /home/xys/Linux-C-Cplus-Project/blog/data/posts.json
```

### Q9: 我想完全手动编辑 JSON 文件，如何验证格式？

**A9**：用 Python 自带的 JSON 检查工具：

```bash
cd /home/xys/Linux-C-Cplus-Project/blog/data
python3 -m json.tool posts.json      > /dev/null && echo "posts OK" || echo "posts ERROR"
python3 -m json.tool categories.json > /dev/null && echo "categories OK" || echo "categories ERROR"
python3 -m json.tool tags.json       > /dev/null && echo "tags OK" || echo "tags ERROR"
```

JSON 语法错误常见原因：
- 数组最后一项后多加了逗号（`]` 前的 `,`）
- 字符串用了单引号 `'` 而非双引号 `"`
- 特殊字符（换行、制表符）没有正确转义

### Q10: 如何让文章中的代码块语法高亮？

**A10**：当前对代码块进行了基础样式处理（深色背景）。要实现语法高亮，引入 highlight.js：

在 `blog/index.html` 的 `<head>` 中添加：

```html
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/styles/atom-one-dark.min.css">
<script src="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/highlight.min.js"></script>
```

然后在 `blog/js/app.js` 的文章渲染完成后调用 `hljs.highlightAll()`。
